## 🔴 CRITICAL BOTTLENECKS (MUST FIX)

### 1. SEQUENTIAL PIPELINE STAGE HANDOFF
- **Impact:** 40 cycles (30.8% of total latency)
- **Root Cause:** FSM wait states block downstream stages
- **Fix:** Remove S_*_W states, add ready/valid handshaking + skid buffers
- **Effort:** MEDIUM (5-7 days)
- **Expected Speedup:** 12-15 cycles (9-11% overall)
- **Priority:** 1 — FOUNDATION (blocks all other optimizations)
- **Files:** optimized_transformer_layer.v, NEW: skid_buffer.v

### 2. SOFTMAX SEQUENTIAL STATE MACHINE
- **Impact:** 25 cycles (19.2% of latency)
- **Root Cause:** 6-state FSM + crude fast_exp() approximation
- **Fix:** Two independent fixes:
  - Replace fast_exp() with exp_lut_256 (TRIVIAL - 5 minutes!)
  - Parallelize tree reduction (1 day)
- **Effort:** LOW-MEDIUM (2 days total)
- **Expected Speedup:** 3-5 cycles (2-4% overall) + accuracy improvement
- **Priority:** 1 — Can do in parallel with #1
- **Files:** parallel_softmax.v

### 3. KV CACHE QUANTIZATION LOOP
- **Impact:** 15 cycles (11.5% of latency)
- **Root Cause:** Two sequential for-loops (min/max search + quantization)
- **Fix:** Parallelize min/max tree + replace division with reciprocal LUT
- **Effort:** MEDIUM (2-3 days)
- **Expected Speedup:** 3-5 cycles (2-4% overall)
- **Priority:** 2 — After pipeline foundation fixed
- **Files:** kv_cache_quantizer.v

### 4. DMA ENGINE MEMORY STALL
- **Impact:** 5-8 cycles per token (scales with model size to 6-11%)
- **Root Cause:** Reactive weight loading (no prefetch orchestration)
- **Fix:** Instantiate existing prefetch_engine.v and wire layer scheduling
- **Effort:** LOW (1-2 days) — Module ALREADY EXISTS!
- **Expected Speedup:** 8-15 cycles hidden latency (6-11%)
- **Priority:** 2 — High ROI for low cost
- **Files:** gpu_system_top_v2.v, layer_pipeline_controller.v
- **Bonus:** Scales with model size; becomes critical for large models

## 🟡 HIGH-IMPACT, LOWER-PRIORITY ITEMS

### 5. SYSTOLIC ARRAY SMALL-MATRIX STALL
- **Impact:** 8 cycles (6.2% of latency)
- **Root Cause:** 4×4 array processing 8D embeddings requires 2 passes
- **Options:**
  - Option A: Increase to 8×8 (4× area cost, 7-cycle reduction)
  - Option B: Weight prefetch tiling (low cost, 2-3 cycle improvement)
- **Effort:** MEDIUM (2-3 days for Option A, 1 day for Option B)
- **Expected Speedup:** 4-7 cycles (3-5% overall)
- **Priority:** 3 — Area/power tradeoff decision
- **Files:** systolic_array.v

### 6. SOFTMAX FIXED-POINT ACCUMULATION
- **Impact:** 2-3 cycles (1.9% of latency)
- **Root Cause:** Narrow accumulators (exp_val: 8-bit, exp_sum: 16-bit)
- **Fix:** Expand to 16-bit exp_val, 21-bit exp_sum
- **Effort:** LOW (1 hour)
- **Expected Speedup:** 1-2 cycles (0.8% overall) + accuracy
- **Priority:** 1 — Trivial, do with Softmax exp LUT fix
- **Files:** parallel_softmax.v

### 7. GQA HEAD REPLICATION MUX PATH
- **Impact:** 3-5 cycles (2.3% of latency)
- **Root Cause:** 256-bit wide data bus creates routing delays
- **Fix:** Add pipeline stage for head replication muxing
- **Effort:** LOW (1 day)
- **Expected Speedup:** 1-2 cycles (0.8% overall) + timing margin
- **Priority:** 2 — Low effort, improves critical path
- **Files:** optimized_transformer_layer.v

### 8. KV CACHE DOUBLE-BUFFER CONFLICTS
- **Impact:** 2-3 cycles (1.5% of latency)
- **Root Cause:** Single KV cache → read-after-write stalls
- **Fix:** Implement ping-pong KV cache (dual RAMs)
- **Effort:** MEDIUM (2 days)
- **Expected Speedup:** 2-3 cycles (1.5% overall)
- **Priority:** 3 — After foundation phases
- **Files:** NEW: kv_cache_pingpong.v

### 9. ACTIVATION COMPRESSION STALL
- **Impact:** 2-3 cycles (1.5% of latency)
- **Root Cause:** Sequential min/max → quantize loop (like KV quantization)
- **Fix:** Parallelize and fuse with GELU output pipeline
- **Effort:** LOW (1 day)
- **Expected Speedup:** 1-2 cycles (0.8% overall)
- **Priority:** 4 — Lower impact
- **Files:** activation_compressor.v

### 10. RECIP LUT LOOKUP LATENCY
- **Impact:** 1-2 cycles (0.8% of latency)
- **Root Cause:** Sequential LUT lookup for reciprocal
- **Fix:** Speculate reciprocal in parallel with exp accumulation
- **Effort:** LOW (1 day, but risky)
- **Expected Speedup:** 0.5-1 cycles (0.4% overall)
- **Priority:** 4 — Speculative, low impact
- **Files:** parallel_softmax.v

## CRITICAL SUCCESS FACTORS

1. **Phase 1 Pipeline Handoff is BLOCKING**
   - All subsequent memory/compute optimizations depend on pipelining
   - Without this, can't hide latency (stages still serialize)
   - Must complete before Phase 2 measurements become meaningful

2. **Test Incrementally (not all-at-once)**
   - Each phase should be measured independently
   - Use cosim_report.txt cycle tracking after each fix
   - Don't accumulate bugs from untested combinations

3. **Preserve Sparsity Benefit**
   - Current 28.7% zero-skip rate = 1.4× speedup multiplier
   - New logic must not break activation sparsity
   - Validate zero-skip rate after each phase (should remain 26-30%)

4. **DMA Prefetch Testing**
   - Current baseline uses small DIM=8 (hides memory stalls naturally)
   - Prefetch benefit only visible with seq_len > 4 tokens
   - Test with multi-token sequences to observe latency hiding

5. **Timing Closure on Critical Path**
   - Several fixes (esp. Softmax, KV quant) impact critical path
   - Synthesis may struggle with wide buses (256-bit GQA data)
   - Run timing analysis after each phase; backend optimization may be needed
