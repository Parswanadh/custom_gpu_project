BASELINE PERFORMANCE (measured from cosim_report.txt):

CRITICAL FINDINGS:

1. PIPELINE STAGE HANDOFF IS THE #1 BOTTLENECK (40 cycles = 30.8% of latency)
   Problem: Six stages execute sequentially with wait states, not overlapped
   Impact: 12-15 cycle reduction possible (9-11% speedup)
   Fix: Implement ready/valid handshaking between stages (Medium effort)
   Files: rtl/integration/optimized_transformer_layer.v

2. SOFTMAX STATE MACHINE IS #2 BOTTLENECK (25 cycles = 19.2% of latency)
   Problem: Six-state FSM uses crude approximation instead of proper LUT
   Impact: 3-5 cycle reduction possible (2-4% speedup)
   Fix: Replace fast_exp() with exp_lut_256 (Low effort, module exists)
   Files: rtl/compute/parallel_softmax.v

3. KV CACHE QUANTIZATION IS #3 BOTTLENECK (15 cycles = 11.5% of latency)
   Problem: Sequential loops for min/max scan then per-element scaling
   Impact: 3-5 cycle reduction possible (2-3% speedup)
   Fix: Parallel min/max tree + fused single-pass quantize (Medium effort)
   Files: rtl/memory/kv_cache_quantizer.v

4. SYSTOLIC ARRAY UNDERUTILIZED (8 cycles = 6.2% of latency)
   Problem: 4×4 array processing 8-dim embeddings requires 2 passes
   Impact: 4-7 cycle reduction (increase array to 8×8, 3-5% speedup)
   Fix: Increase ARRAY_SIZE parameter (Medium effort, ~4× area cost)
   Files: rtl/compute/systolic_array.v

5. DMA WEIGHT LOADING NOT PREFETCHED (hidden bottleneck, scales with seq len)
   Problem: Prefetch engine exists but not wired into gpu_system_top_v2
   Impact: 8-15 cycle latency hiding (6-11% speedup)
   Fix: Wire prefetch_engine.dma_request to command_processor (Low effort)
   Files: rtl/top/gpu_system_top_v2.v, rtl/memory/prefetch_engine.v
