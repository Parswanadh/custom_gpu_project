System Architecture

5 major components working together — from token input to predicted output. Every module hand-designed in Verilog IEEE 1364-2005.

1

AXI Weight Memory

External interface for loading model weights via standard AXI4-Lite bus protocol. 4KB SRAM with control/status registers. Verified write 0xDEADBEEF → read 0xDEADBEEF.

2

Embedding Lookup

Converts token IDs into embedding vectors. Stores both token embeddings (vocabulary) and position embeddings (sequence position).

3

Accelerated Transformer Block

The core computation unit. Pre-LayerNorm architecture with Layer Normalization (×2), KV-Cached Multi-Token Attention, Feed-Forward Network powered by GPU Core pipeline (4 lanes), and residual connections.

4

GPU Multi-Core Pipeline

Configurable compute engine. Exploits zero-value sparsity to skip unnecessary multiplications. Tested standalone at 4 cores × 32 lanes = 128-wide parallel processing.

5

GPT-2 Engine

Top-level orchestrator. Feeds tokens through embedding → N transformer layers → final normalization → argmax → predicted next token.

How It Works

From a simple token ID to a predicted next token — 10 stages of hardware-accelerated inference.

1

Token Input

A token enters the system as a simple integer (e.g., "5")

2

Embedding Lookup

The embedding table converts it to a vector of numbers

3

Layer Normalization

Scales the values to a stable range (mean=0, std=1)

4

Attention Scoring

The attention mechanism scores every past token and produces a weighted context

5

KV Cache Store

This token's Key and Value are stored for future tokens to reference

6

GPU Core FFN

The feed-forward network, powered by the pipelined GPU core, transforms the representation

7

Zero-Skip Detection

Avoids multiplying by zero — saving power on every skipped operation

8

Residual Connections

Preserve the original signal through each layer

9

Final Normalization & Argmax

After all layers, produces the predicted next token

10

Autoregressive Repeat

The process repeats for the next token, with the KV cache growing each time

Pipelined GPU Core

▼

5-stage multiply-accumulate pipeline with configurable parallelism. Verified from gpu_core.v source.

1
Fetch
Read N weights simultaneously
2
Dequant
Scale via config offset
3
Zero Check
Detect zero values
4
ALU
Multiply or skip if zero
5
Writeback
Sum & accumulate

LANES parameter tested with 4 and 32. Pipeline latency: 5 clock cycles to fill, then 1 set of results per cycle. Inside transformer: 1 core × 4 lanes. Standalone benchmark: 4 cores × 32 lanes = 128-wide.

🎯

Zero-Skip Optimization

▼

Detects when a weight or activation is zero and skips the multiplication entirely. ReLU outputs zero for all negative inputs; small weights round to zero after quantization.

Hardware implementation: single combinational NOR gate checks all 8 bits simultaneously within a fraction of the clock period.

Configuration Zero-Skips Source
1-layer, synthetic weights 42 accelerated_gpt2_engine_tb
2-layer, synthetic weights 132 multi_layer_test

🧠

KV-Cached Attention

▼

Implements the Transformer attention mechanism with persistent key-value cache for autoregressive generation. From accelerated_attention.v.

Each token produces Q, K, V via matrix multiplication. Query scores against all past Keys, softmax converts to probabilities, output is weighted sum of past Values. K and V cached to avoid recomputation.

Max sequence length: configurable (default 32). Softmax: 256-entry exp() LUT.

Verified output: Token 1 input [512,128,384,64] → Output [495,135,375,75], zero-skips=2. Source: accelerated_attention_tb.

📊

Softmax Exponential LUT

▼

256 entries: LUT[k] = round(255 × exp(-k/64)). All values generated by Python math.exp() — identical for every hardware deployment.

LUT Index (k) Real Value LUT Output
0 exp(0) = 1.000 255
64 exp(-1.0) = 0.3679 94
128 exp(-2.0) = 0.1353 35
192 exp(-3.0) = 0.0498 13
255 exp(-3.98) = 0.0187 5

Max rounding error: ±1 out of 255 (±0.4%). Replaced earlier linear approximation which was much less accurate.

🔢

Q8.8 Fixed-Point Arithmetic

▼

16-bit integers: 8 bits integer, 8 bits fractional. Simpler hardware than float32 — fewer gates, lower latency, less power.

Property Float32 Q8.8 (Ours)
Total bits 32 16
Memory per weight 4 bytes 2 bytes
Value range ±3.4 × 10³⁸ ±127.996
Precision ~7 digits ~2.4 digits

Store: multiply by 256, keep as int. Add: just add (scale cancels). Multiply: multiply ints, shift right 8.

🔌

AXI4-Lite Memory Interface

▼

Standard bus interface for CPU/DMA to load model weights into GPU on-chip memory. From axi_weight_memory.v.

Protocol: AXI4-Lite. Address width: 16 bits. Data width: 32 bits. Weight memory: 4,096 bytes (4KB) on-chip SRAM.

Address Type Name Description
0x0000–0x0FFF R/W Weight Memory 4KB of 8-bit weights
0x1000 Write Control Bit 0: Start inference
0x1004 Read Status Bit 0: Busy, Bit 1: Done
0x1008 Read Weight Count Number loaded
0x100C Read Zero-Skip Count Total during inference
